Simple test of Google bot's JS capabilities

Have you ever had an SEO question and went to the web in search for answers only to be overcumbered by contradicting opinions? Me too.

This simple (very simple) test will try to answer only 1 question: Will Google bot index a page that has it's robots meta tag set to "index, follow" but dynamically modified to "noindex, nofollow" using JS? Some opinions say that Google will only look at the meta robots tag retrieved from the header response, meaning the source code, and not pay attention to the changes done after page load by JS. Other opinions say that it will take the JS modification into acount as it is getting better at understanding the whole picture by parsing not only the document but JS and CSS external files.

Some documentation

  2. This article has a link to reddit where a Google guy states that robots.txt only takes care of the crawling but not the deindexing. To deindex, you should set your meta tags right and let it crawl. How to Efficiently Deindex Pages
  3. This link explains how disallowing crawling of your page using robots.txt won't neccesarily deindex the page and might still show in search engine results page. If I block Google from crawling a page using a robots.txt disallow directive, will it disappear from search results?
  4. Article showing .htaccess example to set "noindex, nofollow" site wide in case you don't have access to the code and are unable to set the meta robots tag in the code Preventing your site from being indexed, the right way
