Simple test of Google bot's JS understanding

Have you ever had an SEO question and went to the web in search for answers only to be overcumbered by contradicting opinions? Me too.

This simple test will try to answer only 1 question: Will Google bot index a page that has it's robots meta tag set to "index, follow" but dynamically modified to "noindex, nofollow" using JS? Some opinions say that Google will only look at the meta robots tag retrieved from the header response, meaning the source code, and not pay attention to the changes done after page load done by JS. Other opinions say that it will take the JS modification into acount as it is getting better at understanding the whole picture by parsing not only the document but JS and CSS external files.

Will Google bot index my page if...

  1. Control test
  2. Modify robots meta tag with jQuery - It works!
  3. Insert robots meta tag with jQuery - It works!
  4. Modify robots meta tag with pure Javascript - It works!
  5. Insert content with AJAX from own domain and from an external domain - processing...

Will Google bot de-index my page if...

  1. Dissallow page using robots.txt - It didn't work :(
  2. Modify robots meta tag with jQuery - It didn't work :(
